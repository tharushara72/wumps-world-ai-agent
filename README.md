The PEAS FrameworkTo design this agent, we define it using the PEAS descriptor:Performance Measure: +1000 for picking up gold and returning to start; -1000 for falling into a pit or being eaten; -1 per move.Environment: A $4 \times 4$ grid (The World).Actuators: Move (Forward), Turn (Left/Right), Grab (Gold), Shoot (Arrow).Sensors: Percepts including Stench (near Wumpus), Breeze (near Pit), and Glitter (at Gold).2. Knowledge-Based NavigationThe agent functions as a Knowledge-Based Agent. It doesn't just move randomly; it maintains an internal "map" of the world based on its percepts:Inference: If the agent is at (0,0) and detects no Breeze, it infers that (0,1) and (1,0) are Safe.Frontier Exploration: The agent maintains a "Frontier"—a list of known safe cells it has discovered but not yet visited.3. Logic-Based Decision MakingThe agent uses Propositional Logic to determine safety:$B_{x,y} \iff (P_{x,y+1} \lor P_{x,y-1} \lor P_{x+1,y} \lor P_{x-1,y})$(A breeze at (x,y) if and only if there is a pit in an adjacent cell).By proving a cell is "Safe," the agent minimizes risk, a concept known as Reasoning under Uncertainty.🛠️ Tech StackLanguage: Python 3.xVisualization: Matplotlib (for real-time grid rendering)Logic: Custom Rule-Based Inference Engine
